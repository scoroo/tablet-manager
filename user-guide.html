System Overview

The Tablet Management System allows you to control multiple Android tablets from one central PC. Upload APKs, install apps on multiple devices simultaneously, and monitor device status - all from a beautiful web interface.

Features

  • ✅ Auto-discovery of tablets on network
  • ✅ Multiple APK management
  • ✅ Bulk install on multiple devices
  • ✅ Device info detection (model, Android, battery)
  • ✅ Wireless ADB - no USB after setup
  • ✅ QR code enrollment (Android 11+)
  • ✅ No internet required - works offline

Requirements

  • 💻 Windows 10/11 PC
  • 📱 Android tablets (11+ recommended)
  • 🔌 USB cables (data transfer capable)
  • 🌐 Same WiFi network for all devices
  • 📦 Node.js (included in package)

Quick Setup Summary

1
Install Node.js and run npm install
2
Enable Developer Options & USB Debugging on tablet
3
Connect tablet via USB and run enable-wifi-adb.bat
4
Disconnect USB - tablet now works over WiFi
5
Double-click start.bat and open http://localhost:3000
6
Click Detect on your tablet - Done!

PC Setup

Step 1: Install Node.js

Node.js is required to run the server. Download and install the LTS version:

Download Node.js

Choose the LTS version (18.x or 20.x). Install with default settings.

Step 2: Extract the System

Extract the tablet-manager folder to your desired location:

C:\tablet-manager\

Step 3: Install Dependencies

Open PowerShell/Terminal in the tablet-manager folder and run:

npm install

Wait for all packages to install. You'll see a node_modules folder created.

Step 4: Verify ADB

Check that ADB is working:

.\adb\platform-tools-latest-windows\platform-tools\adb.exe version

You should see version information.

3
Turn ON USB Debugging
4

Enable ADB over WiFi

This is the most important step. Do this ONCE per tablet. After this, the tablet will always connect wirelessly.

Method A: Using a USB Hub (Fastest for multiple tablets)

1
Connect all tablets to a USB hub
2
Connect the hub to your PC
3
Double-click enable-all-wifi.bat
4
Wait for "COMPLETE" message
5
Disconnect all USB cables

Method B: One by One (No USB Hub)

Repeat for EACH tablet:

1
Connect tablet via USB cable
2
Double-click enable-wifi-adb.bat
3
Wait for "DONE" message
4
Disconnect USB cable

What You Should See

restarting in TCP mode port: 5555
[DONE] You can now disconnect the USB cable!

Start the Server

Each time you want to use the system:

1
Open the tablet-manager folder
2
Double-click start.bat
3
Select "Run as Administrator" if prompted
4
Keep this window OPEN (closing it stops the server)

Verify Server Started

You should see:

==================================================
✅ Tablet Manager Server Running!
==================================================
📱 Open your browser to: http://localhost:3000
==================================================

Open Web Interface

Open Chrome, Edge, or Firefox and go to:

http://localhost:3000

Access from Other Devices

Find your PC's IP address:

ipconfig

Look for IPv4 Address (e.g., 192.168.100.50)

Other devices on same network can access at:

http://192.168.100.50:3000

Enroll Tab - QR Code Setup

This tab allows you to connect new tablets without a USB cable! Works on Android 11+ devices.

How to Use the Enroll Tab

1
Go to the Enroll Tablets tab in the web interface
2
A QR code will appear on your PC screen
3
On your tablet, open the camera app
4
Scan the QR code on the PC screen
5
The tablet will open a webpage automatically
6
Follow the on-screen instructions

What You Need

  • ✅ Tablet with Android 11 or higher
  • ✅ Tablet camera (to scan QR code)
  • ✅ Both devices on same WiFi network
  • ✅ Wireless Debugging enabled on tablet

On-Tablet Steps (After Scanning QR Code)

1
Go to Settings → Developer Options
2
Enable Wireless Debugging
3
Tap "Pair device with pairing code"
4
Enter the 6-digit code on the webpage
5
Click Pair & Connect
6
Done! Tablet is automatically enrolled

Troubleshooting QR Code

ProblemSolution
QR code not appearingRefresh the page or check server is running
Cannot scan QR codeMake sure tablet camera is working
Page won't openCheck tablet is on same WiFi as PC
Pairing failsRestart Wireless Debugging on tablet
Connection timeoutMake sure no firewall is blocking port 3000

Troubleshooting

ProblemSolution
Tablet not showing in listWait 30 seconds or click "Scan Now" button
Detect shows "Failed"Run "enable-wifi-adb.bat" with USB connected first
"Device offline" messageMake sure tablet is on same WiFi network as PC
QR code not showing in Enroll tabRestart server, check IP address, port 3000
ADB command not foundUpdate ADB path in server.js and batch files
"Connection refused"Run "adb tcpip 5555" again with USB connected
Installation failsCheck APK compatibility and tablet storage space
Tablet shows "unauthorized"Revoke USB debugging and re-approve
Huawei tablet not connectingInstall HiSuite and enable "Allow HiSuite to connect"
Samsung tablet not connectingInstall Samsung USB drivers, use USB-A cable

Quick Fixes

  • Restart Wireless Debugging on tablet
  • Restart the server (close start.bat and reopen)
  • Check both devices are on same WiFi
  • Try a different USB cable (data transfer capable)
  • Disable Windows Firewall temporarily to test
